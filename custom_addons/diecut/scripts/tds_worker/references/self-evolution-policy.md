# Self-Evolution Policy

Self-evolution here means controlled accumulation of reusable knowledge.

## Safe to evolve automatically

- alias memories
- brand profiles
- category profiles
- unresolved pattern logs
- review heuristics
- extraction failure notes
- suggested new param keys for review

## Do not change blindly

- production import behavior for destructive updates
- canonical business semantics without review
- table schema assumptions
- main-field vs spec-field routing for ambiguous values without audit notes

## Good pattern

1. observe a repeated issue
2. capture it in a reviewed artifact
3. propose a reusable rule/profile update
4. use it in future reviewed drafts
5. promote to formal workflow once stable
