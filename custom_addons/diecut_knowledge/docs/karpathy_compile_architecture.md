# 知识管理统一入口与编译层架构

## 目标定位

知识管理是行业知识的主入口和主仓库。PDF、TDS、选型指南、加工经验、客户问答、网页资料和有价值的 AI 对话都先作为 raw source 进入知识管理，再由 AI 生成处理方案，人确认后由解析层和 LLM 编译层生成可维护的 Wiki、FAQ、对比分析和交叉引用。

材料选型库只保存结构化事实数据，例如品牌、型号、厚度、胶系、基材、应用、TDS 参数和选型标签。TDS 文档进入知识管理后，可以额外触发 AI/TDS 抽取，生成结构化草稿，经人工校验后写入材料选型库。

## 模块边界

- `diecut_knowledge`：raw source、解析结果、Wiki/FAQ、交叉引用、编译队列、审核发布、Dify 同步。
- `diecut`：材料型号、分类参数、参数值、选型工作台和结构化数据运维。
- `chatter_ai_assistant`：OpenClaw/Odoo 对话桥梁，用于触发资料解析、TDS 抽取、查询材料和查询知识。
- `OpenClaw`：长上下文和重任务执行器，适合 PDF/TDS 解析、复杂结构化抽取和批量编译。
- `Dify`：消费已发布知识，负责检索和问答，不作为知识生产源。

## 主链路

```text
原始资料
→ 知识管理 / 编译源资料
→ AI 生成处理方案
→ 人工确认
→ PDF / 图片 / 网页解析
→ LLM 编译 Wiki + FAQ + 交叉引用
→ 高置信发布，低置信审核
→ 同步 Dify
→ AI 顾问和 chatter_ai_assistant 检索回答
```

TDS 资料增加一条结构化支线：

```text
TDS raw source
→ 知识管理 / AI 处理方案
→ 人工确认
→ 抽取材料参数
→ AI/TDS 生成结构化草稿
→ 人工校验
→ 入库材料选型库
→ 关联 Wiki 重新编译
```

## 五层漏斗

- 输入层：`编译源资料`，唯一收件箱，先收集不分类。
- 方案层：AI 只提案，说明资料类型、建议动作、风险、目标产物。
- 沉淀层：`Wiki 文章` 和材料结构化库分别保存知识网络和事实数据。
- 输出层：Dify、AI 顾问、chatter_ai_assistant 消费已发布知识和材料事实。
- 治理层：Lint 检查孤立文章、重复内容、缺少引用、参数冲突和过期来源。

## 发布规则

编译文章只有在 `published` 状态才进入 Dify。低置信、缺少来源页码、解析失败、关键参数冲突或内容过短的编译结果进入 `review`，不自动同步。

每次编译都必须记录 `compiled_at`、`compiled_hash`、`compile_confidence`、`compile_risk_level`、来源文件和来源页码。定时任务不仅处理从未编译的资料，也会通过 hash 发现来源或关联材料变更后的过期知识。
