## 一、安全修复

### AI Controller 模型白名单

**文件**：`controllers/ai_advisor.py`

`ai_chat` 和 `save_answer` 接口接收前端传来的 `model` 参数，之前无限制。加了白名单 `ALLOWED_MODELS`，只允许 `diecut.catalog.item` / `diecut.kb.article` / `diecut.kb.qa_ticket` 三个模型，防止用户遍历任意数据。

## 二、功能清单

### F4 异步编译队列（基础架构）

**目标**：点击"AI 编译"立刻返回，后台队列处理不阻塞 UI

**新增文件**：
- `models/kb_compile_job.py` — 编译队列模型，含 `pending → processing → done/failed` 状态机
- `views/kb_compile_job_views.xml` — list/form/search 视图

**修改文件**：
- `models/__init__.py` — 注册新模型
- `models/catalog_item_sync.py` — `action_compile_knowledge()` 改为创建 job 后立即返回通知"已加入队列"
- `security/ir.model.access.csv` — 添加 ACL（用户读写，管理员全权限）
- `views/kb_menu.xml` — 添加菜单"配置 → 编译队列"
- `data/ir_cron.xml` — 新增 cron 每 2 分钟处理队列中的 job（默认启用）

**关键设计**：
- 支持 3 种 job 类型：`catalog_item` / `source_document` / `comparison`
- 优先级排序，支持手动立即执行、取消、重试
- 每个 job 处理完毕后 `env.cr.commit()`，防止长事务

### F2 TDS 一键入库

**目标**：上传 TDS PDF → 一键完成"解析→编译→丰富→lint→同步"全流程

**修改文件**：
- `models/source_document_compile.py` — 新增 `action_one_click_ingest()` 方法
- `views/source_document_compile_views.xml` — header 添加"一键入库"按钮（火箭图标）

**流程**：
```
点"一键入库" → 解析PDF（如未解析）→ 加入编译队列 → 队列处理完成 → enrich → lint → 标记同步
```

### F5 增量编译（品牌综述）

**目标**：编译产品文章后，自动更新品牌综述页

**修改文件**：
- `models/kb_article.py` — `compile_source` 增加 `brand_overview`；新增 `compile_source_brand_id` 字段
- `services/kb_compiler.py` — 新增 `SYSTEM_PROMPT_BRAND_OVERVIEW` 提示词 + `compile_brand_overview(brand_id)` 方法

**设计**：
- `compile_from_item` 末尾自动触发品牌综述编译（`try/except` 包裹，失败不影响主流程）
- 品牌综述会汇总该品牌下所有活跃产品，生成包含产品列表、参数对比表、选型建议的 HTML 文章
- 同一品牌多次编译只会更新同一篇文章（基于 `compile_source_brand_id` 查重）
- 新品牌首次编译时自动创建综述文章

### F3 质量仪表盘

**目标**：知识库健康状态一目了然

**新增文件**：

### F1 流式输出

**目标**：AI 顾问的对话从 blocking 改为 SSE 流式，用户看到 token 逐字出现

**修改文件**：
- `services/dify_client.py` — 新增 `chat_messages_stream()` 生成器方法

  使用 `requests.post(stream=True)` + `response.iter_lines()` 逐行解析 Dify SSE 事件
  支持的事件：`agent_message`（含 token）、`message_end`、`error`、`ping`

- `controllers/ai_advisor.py` — 新增 `type="http"` 路由 `/diecut_knowledge/ai/chat_stream`

  返回 `werkzeug.Response(generator, mimetype="text/event-stream")`
  每次 SSE 事件格式：`data: {"token": "...", "done": false}\n\n`

- `static/src/js/ai_advisor_drawer.js` — `send()` 改用 `fetch()` + `ReadableStream`

  逐 token 追加到 `ui.messages`（OWL 的 useState 驱动视图自动刷新）
  关闭 drawer 时通过 `AbortController` 中断流
  保留 `save_answer` 功能不变（仍用 `rpc`）

## 三、编译流水线现状

```
                              ┌─ 产品编译按钮
                              │     ↓
TDS PDF ─→ 一键入库 ─→ 加入编译队列 ─→ cron 每2分钟
                              │         ↓
                              │     编译(KbCompiler)
                              │         ↓
                              │     丰富(KbEnricher)
                              │         ↓
                              │     lint(KbLinter)
                              │         ↓
                              │     标记同步(pending)
                              │         ↓
                              │     品牌综述(增量)
                              │         ↓
                              │   cron同步到Dify
```

所有编译入口（产品编译、资料编译、对比分析）都已串联 `enrich → lint` 流水线。
