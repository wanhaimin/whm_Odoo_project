# 表格宽度控制说明

## 问题
之前的配置允许调整每列的宽度，但整个表格会自动填满视窗宽度。重启后，表格又会重新填满整个屏幕。

## 解决方案
添加了 CSS 样式来控制表格的总宽度行为：

### 1. 表格宽度设置
- `width: auto !important` - 表格宽度根据内容自动调整
- `table-layout: auto !important` - 使用自动表格布局算法
- `max-width: 100%` - 最大不超过容器宽度

### 2. 最小宽度保护
- **材料清单**: 最小宽度 1200px（保证所有列都能显示）
- **材料类别**: 最小宽度 600px（只有两列，宽度较小）

### 3. 工作原理
现在表格的宽度由以下因素决定：
1. **内容宽度**: 根据每列设置的 width 属性
2. **最小宽度**: 不会小于设定的 min-width
3. **最大宽度**: 不会超过视窗宽度（100%）
4. **用户调整**: 您手动调整的列宽会被保存

## 使用效果

### 刷新前后的行为：
- ✅ 列宽度会保持您调整的值
- ✅ 表格总宽度不会自动填满视窗
- ✅ 如果内容较少，表格会紧凑显示
- ✅ 如果内容较多，表格会扩展但不超过视窗

### 调整列宽：
1. 拖动列边界调整单列的宽度
2. Odoo 会自动保存每列的宽度设置
3. 重启后，每列的宽度保持不变
4. 表格总宽度 = 所有列宽之和（不少于最小宽度）

## 好处
- 📏 **更精确的控制**: 表格不会无意义地占满整个屏幕
- 💾 **持久化**: 您的设置在重启后保持不变
- 👀 **更好的可读性**: 紧凑的布局更易于阅读
- 🎯 **灵活性**: 可以根据需要调整到合适的宽度

## 如果您希望表格填满视窗
如果在某些情况下您希望表格填满整个视窗，可以：
1. 手动拉宽每一列，直到表格填满视窗
2. 或者联系管理员临时移除 CSS 中的宽度限制

## 测试步骤
1. 强制刷新浏览器（Ctrl + Shift + R）
2. 打开材料清单或材料类别
3. 调整列宽到您喜欢的大小
4. 刷新页面或重启 Odoo
5. 验证列宽和表格总宽度都保持不变
