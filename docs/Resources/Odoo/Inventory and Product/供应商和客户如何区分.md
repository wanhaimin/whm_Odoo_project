在 Odoo 的较新版本（V13及以上）中，界面上传统的 “是客户” 和 “是供应商” 两个显式复选框已被移除（隐藏）。Odoo 改为使用 “等级 (Rank)” 机制来根据业务行为自动区分。

1. 区分字段 (数据库层面)
供应商：supplier_rank > 0
客户：customer_rank > 0
2. 怎么输入？(如何定性)
Odoo 的逻辑是：“你在哪里创建，他就是什么身份”，或者 “你把他当什么用，他就是什么”。

方法 A：指定入口创建（推荐）
如果您进入 【采购】APP -> 订单 -> 供应商 菜单，点击“创建”。
在这里创建的联系人，系统会自动将 supplier_rank 设为 1。他就是供应商，能直接在采购单中选到。
方法 B：业务驱动（自动晋升）
如果您在 【联系人】APP 中创建了一个普通人（此时他既不是客户也不是供应商）。
当您第一次对他创建采购订单并确认后，系统会自动将他的 supplier_rank +1。下次您就能在列表里直接看到他了。
遇到“选不到人”怎么办？
由于我们刚才加上了 supplier_rank > 0 的过滤条件：

如果您在“联系人”里新建的人选不到，请尝试去 【采购】->【供应商】 菜单里找到他，随便修改保存一下（或直接在那里新建）。
或者暂时在采购单里直接 输入名字并“创建” 一个新条目，而不是从列表里选。

供应商 (Vendor):
路径：采购 (Purchase) APP -> 订单 (Orders) -> 供应商 (Vendors) 菜单。
在这里创建的联系人，默认身份就是供应商。
客户 (Customer):
路径：销售 (Sales) APP -> 订单 (Orders) -> 客户 (Customers) 菜单。
在这里创建的联系人，默认身份就是客户。
会计 (Accounting/Invoicing) APP 中也有对应的菜单：
客户 -> 客户
供应商 -> 供应商
If they want to edit it on the form, debugging mode is needed to see supplier_rank. Or I can give them a snippet to enable a checkbox if they really want it (by customization), but standard workflow is via Menus.

### 如何录入“客户”与“供应商”
Odoo 通过**菜单入口**来区分录入身份：
1.  **录入“供应商”**：
    *   请进入 **【采购】(Purchase) APP**。
    *   点击顶部菜单 **订单 -> 供应商**。
    *   在此处点击“创建”，该联系人默认就是供应商。
2.  **录入“客户”**：
    *   请进入 **【销售】(Sales) APP**。
    *   点击顶部菜单 **订单 -> 客户**。
    *   在此处点击“创建”，该联系人默认就是客户。
*(如果您在【联系人】主APP里直接创建，它初始是中立身份，直到您给他下了采购单或销售单，系统才会自动标记身份。)*
