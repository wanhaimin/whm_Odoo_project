# Odoo 库存分类与仓库设计方案

> 文档创建时间：2026-02-04
> 核心目的：通过一级分类区分原料和成品，二级和三级区分材质

---

## 📦 Odoo 库存管理核心概念

在 Odoo 中，管理原材料、成品、半成品有 **两个维度**：

| 维度                              | 用途                    | 应用场景                 |
| --------------------------------- | ----------------------- | ------------------------ |
| **产品分类** (`product.category`) | 区分产品的**性质/类型** | 原料 vs 成品 vs 材质     |
| **仓库/库位** (`stock.location`)  | 区分产品的**存放位置**  | 原材料仓、成品仓、生产线 |

---

## 🏗️ 推荐架构设计

### 1️⃣ 产品分类结构

用于**产品定义**时确定其性质：

```
📁 原材料 (category_type=raw)
   ├── 📂 胶带类 (material_code_prefix=JD)
   │   ├── 双面胶带
   │   └── 单面胶带
   └── 📂 泡棉类 (material_code_prefix=PM)
       ├── PE泡棉
       └── EVA泡棉

📁 成品 (category_type=finished)
   ├── 📂 模切件
   └── 📂 冲压件

📁 半成品 (category_type=semi)
   └── 📂 分条料
```

#### 分类字段说明

| 级别     | 用途                 | 关键字段                             |
| -------- | -------------------- | ------------------------------------ |
| **一级** | 区分原料/半成品/成品 | `category_type`: raw/semi/finished   |
| **二级** | 区分材质大类         | `material_code_prefix`: 编码前缀     |
| **三级** | 区分具体规格         | `specification`, `default_thickness` |

---

### 2️⃣ 仓库/库位结构

用于**库存管理**时确定存放位置：

```
🏭 主仓库 (WH)
   ├── 📍 原材料库位 (WH/Stock/Raw)
   │   ├── 胶带区
   │   └── 泡棉区
   ├── 📍 生产线库位 (WH/Stock/Production)
   │   ├── 分条工序
   │   ├── 模切工序
   │   └── 半成品暂存
   └── 📍 成品库位 (WH/Stock/Finished)
       ├── 待检区
       └── 合格品区
```

## 💡 如何使用分类进行统计

### 按分类类型统计库存

```python
# 原材料库存统计
raw_materials = self.env['stock.quant'].search([
    ('product_id.categ_id.category_type', '=', 'raw'),
    ('location_id.usage', '=', 'internal'),
])

# 成品库存统计
finished_goods = self.env['stock.quant'].search([
    ('product_id.categ_id.category_type', '=', 'finished'),
    ('location_id.usage', '=', 'internal'),
])

# 半成品统计（在生产线上）
semi_products = self.env['stock.quant'].search([
    ('product_id.categ_id.category_type', '=', 'semi'),
    ('location_id.name', 'ilike', 'Production'),
])
```

### 按仓库/库位统计

```python
# 原材料仓库存
raw_warehouse = self.env['stock.quant'].search([
    ('location_id.complete_name', 'ilike', 'Raw'),
])

# 成品仓库存
finished_warehouse = self.env['stock.quant'].search([
    ('location_id.complete_name', 'ilike', 'Finished'),
])
```

## ✅ 实施步骤

| 阶段       | 任务           | 说明                               |
| ---------- | -------------- | ---------------------------------- |
| **现在**   | 完善产品分类   | 建立清晰的三级分类结构             |
| **第一步** | 配置库位结构   | 在 库存 → 配置 → 库位 中创建子库位 |
| **第二步** | 建立原材料产品 | 使用 `category_type=raw` 的分类    |
| **第三步** | 配置采购入库   | 默认入库到原材料库位               |
| **第四步** | 配置生产工单   | 领料从原材料库位，入库到成品库位   |
| **第五步** | 建立统计报表   | 按分类+库位维度统计                |
