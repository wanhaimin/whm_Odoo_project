# Odoo 固定资产的采购与入账流程 (无需建库存物料)

很多企业在实施 Odoo 时会有疑问：“如果贵重机器、设备不建在 `product.template`（产品库）里，那我采购部怎么下采购单（PO）？财务怎么付款和入账？”

其实，Odoo 的采购模块（Purchase）和财务明细账（Account Move）非常灵活，**它甚至允许你在不建立任何“实体产品”的情况下，直接采购“服务”或“直接记入某个财务科目”**。

以下是 Odoo 标准且专业的固定资产采购与账务处理流程：

---

## 方案一：建立一个“虚拟的服务型产品”作为马甲（最推荐、最常用）

为了让采购部下发采购单时有一个标准的品名可以选，并且能让财务自动带出分类科目，我们会在产品库（`product.template`）中建立**极少数几个“大类马甲”**，但绝对不为每一台独立的机器建一个产品档案。

### 1. 建立虚拟代币产品
在 Odoo 产品管理中新建几个产品：
*   **产品名称**：如“生产设备采购”、“IT硬件采购”、“办公家具采购”。
*   **产品类型 (Product Type)**：必须选择 **服务 (Service)** 或 **消耗品 (Consumable)**。（绝对不能选“可库存产品 Storable Product”，否则系统会要求你给这些概念性的名字做出入库点数）。
*   **财务科目设置**：在这个虚拟产品的“会计”页签下，其“费用科目”不要选常规的成本科目（如主营业务成本），而是直接指向对应的 **资产科目**（如：`1601 固定资产-机器设备`）或 **资产中转科目**（如：`1604 在建工程-设备待安装`）。

### 2. 采购下单与入账流程
1. **采购下单**：采购员创建采购单（PO），产品选择“生产设备采购”。此时最关键的是：采购员可以在**“描述 (Description)”**字段里，手工打字输入真正的设备型号（如：`雅佳 AK-800 自动模切机，含定制收卷轴`）。
2. **收货环节**：因为产品类型是“服务/消耗品”，所以不需要库管员去仓库里点数扫描入库（资产不需要和原材料混在同一个货架上管理）。
3. **财务账单**：财务根据这笔采购单生成“供应商账单 (Vendor Bill)”。此时系统会自动根据刚才虚拟产品预设的科目，将这笔钱记入资产科目。
4. **生成固定资产卡片**：在 Odoo 的财务(Accounting)模块 -> 资产(Assets) -> 新建资产。根据这张供应商账单的金额和发票日期，建立这台机器真正的“身份证”，并开始设定比如分 60 个月（5年）自动提折旧。

---

## 方案二：直接通过“费用科目”下采购单（更硬核的纯财务玩法）

在 Odoo 15 及以后的版本（当然包括 19），采购单的明细行其实是允许**“不选产品，直接选财务类别”**的结构（部分配置下需开启）。

1. 采购员新建采购单。
2. 在明细行里，**产品 (Product) 这一列留空**，直接在描述框输入：“购买财务部用 Lenovo Thinkpad T14 一台”。
3. 或者是直接在 Odoo 安装了费用/采购预算模块后，将明细行业务绑定到某个**资产科目树**上。
4. 后续财务做账单时，直接将该笔款项凭证挂到“固定资产电脑”账目上，然后去资产模块登记卡片。

*(注：大部分企业觉得方案一保留了一个总概念品名，采购部门更适应这种点击搜索的习惯，所以方案一依然是业界首选。)*

---

## 总结：边界极其清晰的管理双轨制

*   **真正的原材料/卖钱的货物**：在 `product.template` 里面每一个型号建一个码（如 `3M-9448-0.15`，`PET膜-透明-0.5`），系统严防死守它们的进销存数量、货架盈亏。
*   **用来生产用的重机台、桌椅电脑**：在采购端只用一个名叫“生产设备购置”的**大类马甲（服务型）**来骗过必须填品名才能下单的系统限制；把它的真实资料用打字的方式写在采购单描述里，把它的灵魂记在**财务固定资产模块**和**生产维修点检模块**里。

这种解耦的做法不仅能算清账目的每一个铜板，而且能保证你的物资流转系统始终奔跑在最高效的状态！
