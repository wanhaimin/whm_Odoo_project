# 模切原材料成本管理与核算操作手册

## 1. 概述
本手册旨在指导采购及工程人员如何利用 Odoo 模切定制模块进行原材料成本的标准化管理，并为“内部价格核算模型”提供精准的数据支持。核心逻辑是将多样化的供应商报价（按卷、按张、按公斤）统一转化为“面积单价（m²）”，实现核算的标准化。

---

## 2. 核心数据架构
系统通过以下四个原生模型及其扩展字段建立数据底座：

### 2.1 产品模板 (product.template)
*   **物理规格**：宽度 (mm)、长度 (m/mm)、厚度 (mm)、克重 (g/m²)。这些是换算的基准。
*   **成本汇总**：`raw_material_price_per_m2` (单价/m²)。此字段由系统自动计算且物理存储，是核算器的主要抓取对象。
*   **核心字段**：`main_vendor_id` (主要供应商)，决定当前生效的成本价。

### 2.2 供应商价格表 (product.supplierinfo)
*   **原生价格**：`price`。代表供应商给出的原始单位报价（如一卷的价格）。
*   **扩展价格**：`price_per_m2` (单价/m²)、`price_per_kg` (单价/kg)。
*   **单位 (uom.uom)**：作为价格的分母，决定了价格是针对“卷”、“张”还是“公斤”。

---

## 3. 标准化操作流程

### 第一步：完善原材料物理规格
在“原材料库”中打开产品档案，务必准确填写：
1.  **形态**：选择“卷料”或“片料”。
2.  **宽度 (mm)**：影响面积及排版利用率计算。
3.  **长度**：卷料以米为单位，片料以毫米为单位。
4.  **克重/密度**：如需换算公斤单价，此项必填。

### 第二步：录入供应商报价
在产品下方的“供应商价格表”页签中：
1.  新增供应商并输入其原始报价（`price`）。
2.  **查看换算结果**：系统会根据物理规格自动算出该供应商对应的 `price_per_m2`。
3.  **多节点比价**：可录入多家供应商，直接对比平米单价，而非卷价。

### 第三步：锁定主要供应商（关键步骤）
1.  在供应商列表中，点击目标行的 **“设为主要”** 按钮（图标为圆形选中状态）。
2.  **系统联动**：点击后，该供应商的平米单价、交期、起订量会自动同步到产品主档的“总结区域”。
3.  **数据激活**：此时，独立核算模型抓取到的单价将正式更新。

---

## 4. 内部价格核算模型应用

### 4.1 核算逻辑
独立核算模型通过 API 或视图关联直接读取 `product.template` 上的 `raw_material_price_per_m2`。

**核算公式示例：**
> **产品材料成本** = (成品净面积 ÷ 利用率) × 面积单价 (price_per_m2) + 加工费用 + 损耗补偿

### 4.2 业务价值
*   **屏蔽单位干扰**：核算工具不再需要识别供应商是按卷卖还是按公斤卖。
*   **规格自动预警**：原材料宽度变化导致的“排版数”变动，可直接在核算模型中体现。
*   **毛利实时追踪**：原材料价格波动后，核算模型能立即计算出最新的毛利底线，指导销售报价。

---

## 5. 注意事项
*   **换算前提**：如果宽度或长度为 0，平米单价将无法计算。
*   **币种自动折算**：系统支持多币种，核算模型应抓取 `raw_material_currency_id` 以确保汇率正确。
*   **损耗系数**：本模块仅负责“材料单价标准化”，生产过程中的工艺损耗应在“独立核算模型”中通过系数设定。
